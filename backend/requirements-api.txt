# FinSight backend — SLIM deploy dependencies (api sentiment backend).
#
# Use this instead of requirements.txt on low-RAM hosts (Render/Fly free tiers)
# with SENTIMENT_BACKEND=api. It omits torch, transformers, accelerate, and
# sentence-transformers (~2-3GB) — FinBERT runs via the HuggingFace Inference
# API instead of in-process.
#
# Trade-off: the ChromaDB vector store uses SentenceTransformerEmbeddingFunction
# (needs torch), so retrieve_from_vector_store degrades to UNAVAILABLE on this
# build. The router already handles that gracefully (try/except -> UNAVAILABLE),
# so the analysis pipeline still completes — it just skips semantic memory.
# For full parity without torch, switch vector_store.py to chromadb's built-in
# ONNX embedding function (same all-MiniLM-L6-v2, no torch) — future enhancement.
#
# Pins mirror requirements.txt. Keep the two files in sync when bumping shared deps.
